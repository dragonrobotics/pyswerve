occupancy_grid module¶

Implements occupancy grid mapping as a Cythonized extension module.

class occupancy_grid.OccupancyGrid¶

Bases: object

Generate a map from rangefinder sensor data.

The map is represented as a set of uniform grid squares that span the region of interest; each square is assigned a value describing how likely it is to be ‘occupied’ by an object.

This particular implementation uses log-odds to represent occupancy proababilities; that is, instead of storing \(p(occupied)\) directly, the value \(\frac{p(occupied)}{1 - p(occupied)}\) is stored instead. This is for the sake of numerical stability. It should be noted, however, that with this representation positive values represent probabilities greater than 50%, and that negative values represent probabilities less than 50%.

grid¶

2-dimensional float ndarray – The computed map. Each value in this array corresponds to a particular grid square; the value is the log-odds representation of the probability that this grid square is occupied.

scale¶

float – The map scale factor. This is the length of one grid square in reality.

Initialize an occupancy grid map.

Parameters:
  • grid_scale (float) – The scale factor to use for this map.
  • grid_size (integer array-like, shape (2,)) – The size of the region covered by this map, in grid squares.

occupancy_grid.cells_on_line()¶

Iterate over all cells covered by a line segment stretching from p0 to p0+d.

The returned line can optionally be constrained to the rectangle defined by g0 (as the lower-left corner) and g1 (as the upper-right corner). Both corners are included in this rectangle.

Parameters:
  • p0 (ndarray of shape (2,)) – The start point of the line.
  • d (ndarray of shape (2,)) – The displacement of the line along the X and Y axes.
  • g0 (ndarray of shape (2,), optional) – The lower-left corner of the boundary region.
  • g1 (ndarray of shape (2,), optional) – The upper-right corner of the boundary region.
Yields:

integer ndarray, shape (2,) – The next point on the line between p0 and p0+d.

occupancy_grid.get_raycast_distance()¶

Perform a raycast from a point, in a given direction, on a binary grid.

This function computes the distance between a point and the nearest occupied cell in a given direction, up to a specified maximum distance.

Parameters:
  • p0 (float ndarray, shape (2,)) – The start point for the raycast.
  • theta (float) – The direction for the raycast, in a right-handed coordinate system.
  • max_dist (float) – The maximum distance to search out for.
  • grid (2-dimensional integer ndarray) – The grid on which the search is performed; grid cells with values greater than 0 are assumed to be occupied.
Returns:

The distance to the closest occupied cell, or max_dist if no such cell could be found.
Return type:

float
occupancy_grid.list_cells_on_line()¶

Calculate and list all points along the line from p0 to p0+delta.

The returned line is additionally constrained to fall inside the rectangular region between the origin and the boundary point.

Parameters:
  • p0 (float memoryview shape (2,)) – The start point for the line.
  • delta (float memoryview, shape (2,)) – The X and Y displacement of the line’s other endpoint.
  • boundary (int memoryview, shape (2,)) – the upper-right corner of the boundary region, inclusive.
Returns:

A 2-dimensional array containing all grid points covered by the line, ordered from closest to farthest.

integer ndarray, shape (t, 2)
